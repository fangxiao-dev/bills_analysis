# Commenting Conventions

本文件定义新增或修改代码时的注释要求。

## Scope

适用于：
- 新增或修改的函数定义
- 新增或修改的类定义
- 新增脚本
- 枚举定义

## Rules

- 对新增或修改的函数、类定义，以及新增脚本，补充简洁注释。
- 统一优先使用三引号注释；长度按复杂度决定，一句话即可，不必写成大段模板化说明。
- 注释应解释职责、边界或不直观之处，不要机械复述代码字面含义。
- 枚举项使用行尾 `#` 注释说明语义。
- 如果是 Pydantic 定义的枚举类，类本身也需要三引号注释。

## Style Guidance

- 优先保持简洁、具体、可执行。
- 如果文件已有成熟风格，优先与该文件现有风格保持一致。
